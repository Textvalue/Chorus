Choir · Component Build Kit

The exact components that build every screen of Choir.

A team content-OS for B2B SaaS GTM, LinkedIn-first. shadcn/ui is the base for ~90% of the app; a motion component is reached for only where it materially lifts the screen. Linear / Vercel / Notion / Raycast restraint plus tasteful motion — one electric-violet accent on a clean neutral palette, whitespace and hierarchy over decoration. Hand this to engineers: the build is the list of npx shadcn add commands below.

MCP servers — so engineers/agents pull components by name

  • shadcnNatural-language component installs in-session — npx shadcn@latest mcp init --client claude. Free.
  • 21st.dev MagicOptional premium motion pulls. Try Motion-Primitives free equivalents first — claude mcp add 21st-dev …. Browse free; full pulls paid.
  • MobbinGround UI against real shipped flows + mine shipped microcopy. Browser OAuth, no key. Pro-gated (~£8/mo).
  • ReferoClean-premium DESIGN.md calibration (Linear / Vercel / Cursor) + shipped-screen query. Free beta for DESIGN.md copy-paste.

Pin exact versions in .mcp.json — npx -y <pkg>@latest re-resolves every session (supply-chain). Mobbin / Refero are reference-grounding only — study structure and microcopy, never clone an identity.

How to read the kit

Tags, source chips, and the motion marker that appear on every screen row — and the doctrine that governs them.

Source libraries

shadcn/uiThe base for ~90% of the app. Copy-paste source — you own and restyle the code; nothing is npm-locked.
Motion-PrimitivesThe core motion set — free / MIT, spring-physics, owned source. 6 components serve ~30 motion moments via reuse.
AceternityThe surgical accent layer — free tier, 3–4 max in the app interior. Harvest bones, burn skin (recolor + halve speed).
WatermelonFree/MIT layout-skeleton references — strip to skeleton, rewire to Choir metrics. Never ship stock SaaS styling.
ComponentryFree/MIT — at most ONE flourish (Split-Flap on a single hero stat). Most of the catalog is banned (dark-neon).
bespokeBuilt once as owned source on a shadcn atom — the brain-dump recorder + the 7-dot voice-interview stepper.

Row tags

baseA shadcn primitive — the default, no motion library needed.
accentAn Aceternity/registry component used surgically as a single accent. Always optional.
motionAn earned motion moment. Fires once on resolve/view-enter, honors reduced-motion, animates REAL values only.
canonicalA cross-screen decision resolved once and reused everywhere (the segmented control, the dominant row, the pill system).
optionalShip only if it lifts the screen. Default to subtract — the hover-lift + Skeleton are the safe defaults.
free paidEverything recommended is on a free / MIT tier. Paid is flagged explicitly (premium MCP tiers only — no paid components in the build).

The doctrine in one breath

1 electric-violetOne electric-violet accent (#761fff) — primary CTA, active nav/segment pill, focus ring, one KPI/proof highlight, chart target bars, voice dots, the one "human input" dashed card. Green/amber stay semantically separate (verdict, status, confidence) so they never fight the accent.
base 90%shadcn is the base for ~90%. Reach for a motion/registry component only where it materially lifts the screen — one motion moment per viewport, never decoration.
onceFire-once, never loop. Honor prefers-reduced-motion via an SSR-safe project hook. Banned: Aceternity neon/aurora/beams, cursor-particles, dark-mode glows — the "AI slop" register.
